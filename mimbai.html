Definition

What is MIMBAI?

MIMBAI stands for Matching Intelligence for Modern Business & AI — Oslo Meet's proprietary B2B matching engine.

MIMBAI is not a directory, a CRM, or a generic recommendation system. It is an intelligent matching layer built specifically for B2B contexts, designed to surface real strategic compatibility between companies — not just keyword overlap.

Every company that joins Oslo Meet is analyzed by MIMBAI. Its profile is converted into high-dimensional semantic vectors, compared against other companies, and assigned a compatibility score based on industry fit, shared goals, technology synergies, and market alignment.

MIMBAI runs continuously in the background — learning from interactions, refining scores, and surfacing new matches as the Oslo Meet network grows.

Process

How MIMBAI Works

A 4-step pipeline from raw company data to high-confidence match recommendations.

// 01

Profile Ingestion

Company data — sector, goals, technologies, markets, collaboration preferences — is structured into semantic tags and normalized business vectors.

// 02

Embedding Generation

Text fields are converted into high-dimensional embeddings using the OpenAI API. Each company becomes a point in a shared semantic space.

// 03

Compatibility Scoring

MIMBAI computes a 0–100% compatibility score using weighted signals: industry fit, strategic goals, tech synergy, market overlap, and collaboration intent.

// 04

Match Delivery

Top matches are surfaced in each company's dashboard feed. The AI Concierge can draft personalized introductions based on match context.

Architecture

Technical Foundation

MIMBAI is built on a modern, GDPR-compliant stack designed for performance, privacy, and scalability.

Semantic Embeddings

OpenAI API

Company profiles are converted into high-dimensional vector representations using OpenAI's embedding models, enabling deep semantic comparison beyond keyword matching.

Vector Storage

Supabase + pgvector

All embeddings and company data are stored in a GDPR-compliant, EU-hosted Supabase database with Row-Level Security enforcing strict data access policies.

Scoring Engine

Rule-based + ML

A hybrid scoring engine combines cosine similarity on embeddings with rule-based weights for sector-specific signals, producing a single interpretable compatibility score.

GDPR Compliance

EU/EEA standard

Data is encrypted at rest and in transit. Users can opt out of AI processing, request human review of decisions, or export/delete their data at any time.

Realtime Updates

Supabase Realtime

Match feeds and messages update in real time via WebSocket subscriptions. New companies trigger automatic re-scoring across the existing member base.

AI Concierge

LLM-powered drafts

Once a high-confidence match is found, the AI Concierge generates a personalized introduction draft. Users always approve before any message is sent.

Privacy by design. Your business data is processed only for matching within Oslo Meet. MIMBAI never uses company profiles to train public AI models, never sells data to third parties, and is fully auditable by registered members under GDPR Article 22.
